# 架构说明

## 流程
1. 用户在侧边栏输入 API Key 与 Base URL。
2. （可选）上传 PDF，按页解析为文档。
3. 文档分块（带重叠）并写入本地 Chroma 向量库。
4. 从文档内容生成结构化 JSON 摘要。
5. 问答时检索 Top-K 片段，构造上下文提示。
6. 有检索结果时回答附带来源引用。

## 组件
- app.py：UI、会话状态、交互流程
- rag_engine.py：数据处理、向量检索、LLM 调用
- Chroma：本地持久化向量库

## 数据与可观测性
- 侧边栏展示检索日志，同时打印到控制台。
- 当存在检索结果时，回答末尾会追加来源。
